# FE-06: Streaming AI Chat Interface

## Overview
This assignment implements a production-quality, streaming AI chat interface into the Next.js capstone project, utilizing the Vercel AI SDK and the FastRouter AI gateway.

## Features Implemented
- **AI SDK Configuration (`src/lib/ai/config.ts`)**: Secure server-side configuration using `fastrouter.ai` and `anthropic/claude-haiku-4.5`.
- **API Route (`src/app/api/chat/route.ts`)**: Built with `@ai-sdk/react` 5.0+ and `ai` 7.x standards, safely converting UIMessages to ModelMessages and utilizing `streamText()` with a `toUIMessageStreamResponse()`.
- **Robust UI Components**:
  - `ChatInterface.tsx`: The primary container using `useChat` hook, maintaining manual input state and sending messages through `DefaultChatTransport`.
  - `ChatComposer.tsx`: A robust textarea with auto-resizing, handling Enter-to-submit and Shift-Enter for newlines.
  - `MessageList.tsx` & `MessageBubble.tsx`: Handles message presentation and conditional rendering for AI vs user, safely iterating through typed message `parts`.
  - `ThinkingIndicator.tsx`: Animated bouncing dots indicating generation state.
  - `JumpToLatestButton.tsx`: Manual scroll-down interaction when the user scrolls away from the bottom stream.
- **Auto-Scroll Behavior**: Managed via React `useEffect` and refs using standard DOM calculations (`scrollHeight`, `scrollTop`) to determine whether pinning to the bottom should pause or resume.
- **Storage Persistence**: Uses `localStorage` keyed with `fe06-chat-messages-v1` to persist user sessions across reloads, wrapped in a `useEffect` to safely handle SSR constraints.

## Challenges & Solutions
1. **AI SDK Version API Changes**: We identified that `@ai-sdk/react` and `ai` were using v5+ paradigms where `maxSteps` was removed in favor of `stopWhen`, `toDataStreamResponse` replaced with `toUIMessageStreamResponse`, and `useChat` no longer exported `input` or `handleInputChange`. This required manually managing the input state with `useState` and switching to `sendMessage()`.
2. **Types & Message Properties**: The `content` property on UIMessages is replaced with `parts` arrays in the latest AI SDK. This required iterating over `message.parts` to render text nodes safely and to perform character limits server-side without TypeScript complaining about missing properties.

## Security
The `ANTHROPIC_API_KEY` exists purely server-side via `.env.local` or Vercel Environment Variables. The frontend UI solely interacts with `/api/chat`, keeping the gateway URL and prompt context completely secure from client introspection.
